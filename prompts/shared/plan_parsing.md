# Phase 1: Plan Parsing

You are evaluating a plan document. Read it carefully and extract the following:

## Plan Under Review

**Path:** {{PLAN_PATH}}

<plan>
{{PLAN_CONTENT}}
</plan>

## Your Task

1. **Goals:** What is this plan trying to accomplish? List the main objectives.
2. **Steps:** Summarize each major step or task in the plan (one line each).
3. **File References:** List every file the plan mentions creating, modifying, or testing.
4. **Assumptions:** What does the plan take for granted? What is implied but not stated?
5. **Dependencies:** What existing code, APIs, or systems does this plan depend on?

Present your summary and ask the user to confirm it is accurate before proceeding.
